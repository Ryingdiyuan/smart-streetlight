# [OPEN] dashboard-no-data

## 现象
- 用户反馈软件前端“还是没有任何数据”
- 之前已修复过 `/api/devices` 因 `sensors` 表缺字段导致的 500
- 当前需要确认是哪些接口仍在失败，还是前端鉴权/聚合接口返回空

## 假设
1. 总览页和地图页依赖的不止 `/api/devices`，仍有其他接口因为旧库结构不兼容而报 500
2. 前端当前登录态异常，导致关键业务接口返回 401，页面统一表现为无数据
3. 后端聚合统计接口能返回，但数据库中的相关明细表为空，导致大盘展示为 0
4. 地图页单独依赖的接口或数据映射仍引用了重构前字段，导致页面拿到数据却无法显示

## 计划
- 确认前端总览页/地图页实际调用的接口
- 逐个验证这些接口的返回情况
- 结合后端日志定位仍在失败的具体接口和具体错误
- 基于证据做最小修复，并再次验证
