# 数据库设计

## 设计目标

数据库用于保存智慧路灯系统的设备、光照数据、阈值配置、控制日志和告警日志。Day 4 在 Day 3 `devices` 表基础上新增 `light_data` 表，用 HTTP 接口模拟设备上报光照数据，并支持最新数据和历史数据查询。

## 本地开发配置

- Host：`127.0.0.1`
- Port：`3307`
- Database：`smart_streetlight`
- User：`root`
- Password：通过后端 `.env` 的 `MYSQL_PASSWORD` 配置，不写入代码仓库
- Charset：`utf8mb4`

测试前先在 MySQL 中创建数据库：

```sql
CREATE DATABASE IF NOT EXISTS smart_streetlight
DEFAULT CHARSET utf8mb4
COLLATE utf8mb4_unicode_ci;
```

当前开发阶段使用 `Base.metadata.create_all(bind=engine)` 在后端启动时创建已声明的表，暂不引入 Alembic。

## 表结构

### devices

路灯设备基础信息表。Day 3 已实现。

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| id | int | primary key, auto increment | 设备 ID |
| device_code | varchar(64) | unique, not null | 设备编码，例如 `SL-001` |
| device_name | varchar(100) | not null | 设备名称，例如 `一号路灯` |
| location | varchar(255) | nullable | 安装位置，例如 `校门口` |
| status | varchar(20) | not null, default `offline` | 设备状态，例如 `online`、`offline` |
| created_at | datetime | not null | 创建时间 |
| updated_at | datetime | not null | 更新时间 |

### light_data

设备光照和路灯状态上报数据表。Day 4 已实现。

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| id | int | primary key, auto increment | 光照数据 ID |
| device_id | int | not null, foreign key | 关联 `devices.id` |
| light_intensity | int | not null | 光照强度，例如 `120` |
| lamp_status | varchar(20) | not null | 路灯状态，例如 `on`、`off` |
| voltage | float | nullable | 电压，例如 `220.5` |
| reported_at | datetime | not null | 设备上报时间 |
| created_at | datetime | not null | 后端入库时间 |

### threshold_configs

自动联动阈值配置表。Day 6 再实现。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | bigint | 主键 |
| device_id | bigint | 关联 `devices.id` |
| low_threshold | int | 低于该值自动开灯 |
| high_threshold | int | 高于该值自动关灯 |
| enabled | tinyint | 是否启用自动控制 |
| updated_at | datetime | 更新时间 |

### control_logs

控制指令日志表。Day 7 再实现。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | bigint | 主键 |
| device_id | bigint | 关联 `devices.id` |
| command | varchar(50) | 指令：`TURN_ON`、`TURN_OFF`、`SET_BRIGHTNESS` |
| source | varchar(20) | 来源：`manual`、`auto` |
| result | varchar(20) | 结果：`pending`、`success`、`failed` |
| request_payload | json | 下发内容 |
| reply_payload | json | 设备回复内容 |
| created_at | datetime | 创建时间 |

### alarm_logs

告警日志表。Day 8 再实现。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | bigint | 主键 |
| device_id | bigint | 关联 `devices.id` |
| alarm_type | varchar(50) | 告警类型：`offline`、`light_abnormal`、`command_failed` |
| alarm_level | varchar(20) | 告警等级：`info`、`warning`、`critical` |
| alarm_content | varchar(500) | 告警内容 |
| handled | tinyint | 是否已处理 |
| handled_at | datetime | 处理时间 |
| created_at | datetime | 创建时间 |
