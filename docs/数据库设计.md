# 数据库设计

## 设计目标

数据库优先支持第一阶段演示链路：设备注册、光照数据上报、路灯状态记录、阈值配置、控制指令日志、告警日志。数据库使用 MySQL，建议字符集使用 `utf8mb4`。

本地开发配置：

- Host：`127.0.0.1`
- Port：`3307`
- Database：`smart_streetlight`
- User：`root`
- Password：通过后端 `.env` 的 `MYSQL_PASSWORD` 配置

## 表结构初稿

### devices

路灯设备基础信息表。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | bigint | 主键 |
| device_code | varchar(64) | 设备编码，唯一，例如 `SL-001` |
| device_name | varchar(100) | 设备名称 |
| location | varchar(255) | 安装位置 |
| status | varchar(20) | 在线状态：`online`、`offline` |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

### light_data

设备光照和路灯状态上报数据表。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | bigint | 主键 |
| device_id | bigint | 关联 `devices.id` |
| light_intensity | int | 光照强度 |
| lamp_status | varchar(20) | 路灯状态：`on`、`off` |
| reported_at | datetime | 设备上报时间 |
| created_at | datetime | 入库时间 |

### threshold_configs

自动联动阈值配置表。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | bigint | 主键 |
| device_id | bigint | 关联 `devices.id` |
| low_threshold | int | 低于该值自动开灯 |
| high_threshold | int | 高于该值自动关灯 |
| enabled | tinyint | 是否启用自动控制 |
| updated_at | datetime | 更新时间 |

### control_logs

控制指令日志表。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | bigint | 主键 |
| device_id | bigint | 关联 `devices.id` |
| command | varchar(50) | 指令：`TURN_ON`、`TURN_OFF`、`SET_BRIGHTNESS` |
| source | varchar(20) | 来源：`manual`、`auto` |
| result | varchar(20) | 结果：`pending`、`success`、`failed` |
| request_payload | json | 下发内容 |
| reply_payload | json | 设备回复内容 |
| created_at | datetime | 创建时间 |

### alarm_logs

告警日志表。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | bigint | 主键 |
| device_id | bigint | 关联 `devices.id` |
| alarm_type | varchar(50) | 告警类型：`offline`、`light_abnormal`、`command_failed` |
| alarm_level | varchar(20) | 告警等级：`info`、`warning`、`critical` |
| alarm_content | varchar(500) | 告警内容 |
| handled | tinyint | 是否已处理 |
| handled_at | datetime | 处理时间 |
| created_at | datetime | 创建时间 |

## 后续优化

1. 为 `light_data.device_id, reported_at` 增加联合索引，支持历史曲线查询。
2. 为 `alarm_logs.handled, created_at` 增加索引，支持告警列表筛选。
3. 真实设备接入后，可增加设备固件版本、经纬度、最后心跳时间等字段。
