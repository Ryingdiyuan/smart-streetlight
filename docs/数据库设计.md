# 数据库设计

## 设计目标

数据库用于保存智慧路灯节能系统的用户、设备、光照数据、阈值配置、控制日志和告警日志。

当前阶段已经包含：

1. `users`：登录用户和角色信息。
2. `devices`：设备基础信息。
3. `light_data`：设备光照和状态上报数据。
4. `threshold_configs`：自动判断阈值配置。
5. `control_logs`：手动控制指令日志。
6. `alarm_logs`：设备离线等告警日志。

## 本地开发配置

- Host：`127.0.0.1`
- Port：`3307`
- Database：`smart_streetlight`
- User：`root`
- Password：通过后端 `.env` 的 `MYSQL_PASSWORD` 配置，不写入代码仓库
- Charset：`utf8mb4`

测试前先在 MySQL 中创建数据库：

```sql
CREATE DATABASE IF NOT EXISTS smart_streetlight
DEFAULT CHARSET utf8mb4
COLLATE utf8mb4_unicode_ci;
```

当前开发阶段使用 `Base.metadata.create_all(bind=engine)` 在后端启动时创建已声明的表，暂不引入 Alembic。

## 表结构

### users

系统登录用户表。Day 13 新增，用于后端登录认证和角色权限基础。

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| id | int | primary key, auto increment | 用户 ID |
| username | varchar(64) | unique, not null | 用户名 |
| password_hash | varchar(255) | not null | 密码哈希，不保存明文密码 |
| role | varchar(20) | not null, default `viewer` | 用户角色：`admin`、`operator`、`viewer` |
| is_active | boolean | not null, default `true` | 是否启用 |
| created_at | datetime | not null | 创建时间 |
| updated_at | datetime | not null | 更新时间 |

说明：

- `POST /api/auth/init-admin` 只在 `users` 表为空时创建第一个管理员。
- 登录成功后返回 JWT，不返回 `password_hash`。
- Day 13 只新增认证基础，不强制保护旧业务接口。

### devices

路灯设备基础信息表。

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| id | int | primary key, auto increment | 设备 ID |
| device_code | varchar(64) | unique, not null | 设备编码，例如 `SL-001` |
| device_name | varchar(100) | not null | 设备名称，例如 `一号路灯` |
| location | varchar(255) | nullable | 安装位置，例如 `校门口` |
| status | varchar(20) | not null, default `offline` | 设备状态，例如 `online`、`offline` |
| last_heartbeat_at | datetime | nullable | 设备最后一次心跳时间 |
| created_at | datetime | not null | 创建时间 |
| updated_at | datetime | not null | 更新时间 |

如果本地 MySQL 中已经存在旧版 `devices` 表，需要手动补充字段：

```sql
ALTER TABLE devices ADD COLUMN last_heartbeat_at DATETIME NULL;
```

### light_data

设备光照和路灯状态上报数据表。

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| id | int | primary key, auto increment | 光照数据 ID |
| device_id | int | not null, foreign key | 关联 `devices.id` |
| light_intensity | int | not null | 光照强度，例如 `120` |
| lamp_status | varchar(20) | not null | 路灯状态，例如 `on`、`off` |
| voltage | float | nullable | 电压，例如 `220.5` |
| reported_at | datetime | not null | 设备上报时间 |
| created_at | datetime | not null | 后端入库时间 |

### threshold_configs

设备自动判断阈值配置表。

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| id | int | primary key, auto increment | 阈值配置 ID |
| device_id | int | unique, not null, foreign key | 关联 `devices.id`，一个设备一条配置 |
| low_threshold | int | not null, default `100` | 低阈值，低于该值建议开灯 |
| high_threshold | int | not null, default `300` | 高阈值，高于该值建议关灯 |
| enabled | boolean | not null, default `true` | 是否启用自动判断 |
| updated_at | datetime | not null | 更新时间 |

### control_logs

控制指令日志表。

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| id | int | primary key, auto increment | 控制日志 ID |
| device_id | int | not null, foreign key | 关联 `devices.id` |
| command | varchar(50) | not null | 控制命令：`TURN_ON`、`TURN_OFF`、`SET_BRIGHTNESS` |
| source | varchar(20) | not null, default `manual` | 来源，当前为 `manual` |
| result | varchar(20) | not null | 结果：`success`、`failed`、`skipped` |
| request_payload | json | nullable | 控制请求内容 |
| reply_payload | json | nullable | 设备回复内容，当前暂为空 |
| created_at | datetime | not null | 创建时间 |

### alarm_logs

告警日志表。Day 8 主要用于记录设备离线告警。

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| id | int | primary key, auto increment | 告警 ID |
| device_id | int | not null, foreign key | 关联 `devices.id` |
| alarm_type | varchar(50) | not null | 告警类型，例如 `offline` |
| alarm_level | varchar(20) | not null, default `warning` | 告警等级，例如 `warning` |
| alarm_content | varchar(500) | not null | 告警内容 |
| handled | boolean | not null, default `false` | 是否已处理 |
| handled_at | datetime | nullable | 处理时间 |
| created_at | datetime | not null | 创建时间 |

离线告警规则：

1. 设备心跳通过 MQTT `status` 主题上报。
2. 后端记录 `devices.last_heartbeat_at`。
3. APScheduler 定时检查 `online` 设备。
4. 如果最后心跳时间超过 `DEVICE_OFFLINE_SECONDS`，则设备状态改为 `offline`。
5. 如果该设备没有未处理的 `offline` 告警，则新增一条告警。
