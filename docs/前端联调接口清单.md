# 前端联调接口清单

## 联调基础信息

- 后端基础地址：`http://127.0.0.1:8000`
- API 前缀：`/api`
- Swagger 文档：`http://127.0.0.1:8000/docs`
- 前端本地开发地址：`http://localhost:5173`
- 备用前端地址：`http://127.0.0.1:5173`

当前后端 CORS 已允许以上两个前端本地开发地址。

## 前端页面建议对应关系

| 前端页面或组件 | 建议调用接口 |
| --- | --- |
| 设备列表页 | `GET /api/devices` |
| 设备详情页 | `GET /api/devices/{id}` |
| 新增设备弹窗 | `POST /api/devices` |
| 编辑设备弹窗 | `PUT /api/devices/{id}` |
| 光照曲线 | `GET /api/devices/{device_id}/light-history` |
| 实时状态卡片 | `GET /api/devices/{device_id}/latest-light` |
| 阈值设置弹窗 | `GET /api/devices/{device_id}/threshold`、`PUT /api/devices/{device_id}/threshold` |
| 手动控制按钮 | `POST /api/devices/{device_id}/commands` |
| 控制日志列表 | `GET /api/devices/{device_id}/commands` |
| 告警列表 | `GET /api/alarms` |
| 告警处理按钮 | `PUT /api/alarms/{alarm_id}/handle` |
| 登录页 | `POST /api/auth/login` |
| 当前用户信息 | `GET /api/auth/me` |

## 登录认证

Day 13 新增后端认证接口。当前旧业务接口尚未强制要求 Token，后续逐步接入权限控制时，前端需要在请求 Header 中携带：

```text
Authorization: Bearer <access_token>
```

### POST /api/auth/init-admin

用途：开发或演示环境初始化第一个管理员。

请求体示例：

```json
{
  "username": "admin",
  "password": "123456"
}
```

响应体示例：

```json
{
  "id": 1,
  "username": "admin",
  "role": "admin",
  "is_active": true
}
```

常见错误码：

- `400`：系统已存在用户，不能重复初始化管理员。

### POST /api/auth/login

用途：登录并获取 token。

请求体示例：

```json
{
  "username": "admin",
  "password": "123456"
}
```

响应体示例：

```json
{
  "access_token": "jwt_token_string",
  "token_type": "bearer",
  "user": {
    "id": 1,
    "username": "admin",
    "role": "admin"
  }
}
```

前端接入建议：

- 登录成功后保存 `access_token`。
- 后续需要鉴权的请求添加 `Authorization: Bearer <access_token>`。
- 不要把密码、Token 写死在前端代码中。

常见错误码：

- `401`：用户名或密码错误。
- `403`：用户被禁用。

### GET /api/auth/me

用途：根据 token 获取当前用户。

Header：

```text
Authorization: Bearer <access_token>
```

响应体示例：

```json
{
  "id": 1,
  "username": "admin",
  "role": "admin",
  "is_active": true
}
```

常见错误码：

- `401`：Token 缺失、无效、过期或用户不存在。
- `403`：用户被禁用。

## 设备管理

### GET /api/devices

用途：获取设备列表。

请求参数：无。

响应示例：

```json
[
  {
    "id": 1,
    "device_code": "SL-001",
    "device_name": "一号路灯",
    "location": "校门口",
    "status": "online",
    "last_heartbeat_at": "2026-07-03T10:00:00"
  }
]
```

常见错误码：

- `500`：数据库连接异常或表结构未同步。

### GET /api/devices/{id}

用途：获取设备详情。

路径参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| id | integer | 设备 ID |

常见错误码：

- `404`：设备不存在。

### POST /api/devices

用途：新增设备。

请求体示例：

```json
{
  "device_code": "SL-001",
  "device_name": "一号路灯",
  "location": "校门口",
  "status": "offline"
}
```

响应体示例：

```json
{
  "id": 1,
  "device_code": "SL-001",
  "device_name": "一号路灯",
  "location": "校门口",
  "status": "offline",
  "last_heartbeat_at": null
}
```

常见错误码：

- `400`：设备编码已存在。
- `422`：请求体字段缺失或类型错误。

### PUT /api/devices/{id}

用途：编辑设备信息。

请求体示例：

```json
{
  "device_name": "一号路灯-东门",
  "location": "东门入口",
  "status": "online"
}
```

常见错误码：

- `400`：设备编码重复。
- `404`：设备不存在。

### DELETE /api/devices/{id}

用途：删除设备。

响应体示例：

```json
{
  "message": "删除成功"
}
```

常见错误码：

- `404`：设备不存在。

## 光照数据

### POST /api/devices/{device_id}/light-data

用途：模拟设备通过 HTTP 上报光照数据。

请求体示例：

```json
{
  "light_intensity": 120,
  "lamp_status": "off",
  "voltage": 220.5,
  "reported_at": "2026-07-03T10:00:00"
}
```

响应体示例：

```json
{
  "id": 1,
  "device_id": 1,
  "light_intensity": 120,
  "lamp_status": "off",
  "voltage": 220.5,
  "reported_at": "2026-07-03T10:00:00",
  "created_at": "2026-07-03T10:00:01",
  "suggested_action": "TURN_ON"
}
```

常见错误码：

- `404`：设备不存在。

### GET /api/devices/{device_id}/latest-light

用途：获取设备最新光照数据，用于实时状态卡片。

常见错误码：

- `404`：设备不存在或暂无光照数据。

### GET /api/devices/{device_id}/light-history

用途：获取设备历史光照数据，用于曲线图。

查询参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| limit | integer | 默认 `20`，范围 `1-200` |
| start_time | datetime | 可选，开始时间 |
| end_time | datetime | 可选，结束时间 |

响应体示例：

```json
[
  {
    "id": 1,
    "device_id": 1,
    "light_intensity": 120,
    "lamp_status": "off",
    "voltage": 220.5,
    "reported_at": "2026-07-03T10:00:00",
    "created_at": "2026-07-03T10:00:01"
  }
]
```

## 阈值配置

### GET /api/devices/{device_id}/threshold

用途：获取设备阈值配置。

响应体示例：

```json
{
  "id": 1,
  "device_id": 1,
  "low_threshold": 100,
  "high_threshold": 300,
  "enabled": true,
  "updated_at": "2026-07-03T10:00:00"
}
```

常见错误码：

- `404`：设备不存在。

### PUT /api/devices/{device_id}/threshold

用途：保存设备阈值配置。

请求体示例：

```json
{
  "low_threshold": 100,
  "high_threshold": 300,
  "enabled": true
}
```

常见错误码：

- `400`：`low_threshold` 必须小于 `high_threshold`。
- `404`：设备不存在。

## 控制命令

### POST /api/devices/{device_id}/commands

用途：前端手动控制按钮调用。

请求体示例：

```json
{
  "command": "TURN_ON"
}
```

```json
{
  "command": "SET_BRIGHTNESS",
  "brightness": 80
}
```

响应体示例：

```json
{
  "id": 1,
  "device_id": 1,
  "command": "TURN_ON",
  "source": "manual",
  "result": "success",
  "request_payload": {
    "command": "TURN_ON",
    "source": "manual",
    "timestamp": "2026-07-03 10:00:00"
  },
  "reply_payload": null,
  "created_at": "2026-07-03T10:00:00"
}
```

常见错误码：

- `400`：非法命令或亮度参数错误。
- `404`：设备不存在。

### GET /api/devices/{device_id}/commands

用途：查看设备控制日志。

查询参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| limit | integer | 默认 `20`，范围 `1-200` |

## 告警管理

### GET /api/alarms

用途：查询告警列表。

查询参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| handled | boolean | 可选，按是否处理过滤 |
| alarm_type | string | 可选，例如 `offline` |
| limit | integer | 默认 `20`，范围 `1-200` |

响应体示例：

```json
[
  {
    "id": 1,
    "device_id": 1,
    "alarm_type": "offline",
    "alarm_level": "warning",
    "alarm_content": "设备 SL-001 已离线",
    "handled": false,
    "handled_at": null,
    "created_at": "2026-07-03T10:05:00"
  }
]
```

### GET /api/alarms/{alarm_id}

用途：查询告警详情。

常见错误码：

- `404`：告警不存在。

### PUT /api/alarms/{alarm_id}/handle

用途：标记告警已处理。

常见错误码：

- `404`：告警不存在。

## 统一错误码

| 状态码 | 前端处理建议 |
| --- | --- |
| 400 | 显示后端返回的参数错误原因，例如设备编码重复、阈值配置不合法 |
| 401 | 跳转登录页或提示重新登录 |
| 403 | 提示当前账号无权限或账号已禁用 |
| 404 | 显示资源不存在或暂无数据提示 |
| 422 | 检查请求体字段名、字段类型、必填字段 |
| 500 | 提示后端服务异常，并查看后端终端日志 |
