## 手动控制

控制日志保存到 MySQL 的 `control_logs` 表。Day 7 支持手动下发控制命令，并在 `MQTT_ENABLED=true` 时向 MQTT command topic 发布消息。

### 下发控制命令

接口：

```text
POST /api/devices/{device_id}/commands
```

请求示例：

```json
{
  "command": "TURN_ON"
}
```

```json
{
  "command": "TURN_OFF"
}
```

```json
{
  "command": "SET_BRIGHTNESS",
  "brightness": 80
}
```

说明：

- `command` 只允许：`TURN_ON`、`TURN_OFF`、`SET_BRIGHTNESS`。
- `SET_BRIGHTNESS` 必须传 `brightness`。
- `brightness` 范围为 `0-100`。
- `source` 固定为 `manual`。
- `MQTT_ENABLED=false` 时不发布 MQTT，日志结果为 `skipped`。
- `MQTT_ENABLED=true` 且发布成功时，日志结果为 `success`。
- MQTT 发布失败时，日志结果为 `failed`。

响应示例：

```json
{
  "id": 1,
  "device_id": 1,
  "command": "TURN_ON",
  "source": "manual",
  "result": "success",
  "request_payload": {
    "command": "TURN_ON",
    "source": "manual",
    "timestamp": "2026-07-03 10:00:00"
  },
  "reply_payload": null,
  "created_at": "2026-07-03T10:00:00"
}
```

### 查询控制日志

接口：

```text
GET /api/devices/{device_id}/commands
```

查询参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| limit | integer | 返回条数，默认 `20`，范围 `1-200` |

说明：

- 按 `created_at` 倒序返回。
- 设备不存在返回 `404`。
