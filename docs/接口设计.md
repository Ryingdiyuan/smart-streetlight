## 设备管理

Day 2 使用 Python 内存字典临时保存设备数据，服务重启后数据会丢失。Day 3 再改为 MySQL + SQLAlchemy 持久化存储。

### 设备字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | integer | 设备 ID，由后端生成 |
| device_code | string | 设备编码，唯一，例如 `SL-001` |
| device_name | string | 设备名称，例如 `一号路灯` |
| location | string | 安装位置，例如 `校门口` |
| status | string | 设备状态，例如 `online`、`offline` |

### 接口列表

| 方法 | 路径 | 说明 | 当前状态 |
| --- | --- | --- | --- |
| GET | `/devices` | 查询设备列表 | 已实现 |
| GET | `/devices/{id}` | 查询单个设备详情 | 已实现 |
| POST | `/devices` | 新增设备 | 已实现 |
| PUT | `/devices/{id}` | 修改设备信息，支持部分字段 | 已实现 |
| DELETE | `/devices/{id}` | 删除设备 | 已实现 |

### 新增设备

请求：

```json
{
  "device_code": "SL-001",
  "device_name": "一号路灯",
  "location": "校门口",
  "status": "offline"
}
```

响应：

```json
{
  "device_code": "SL-001",
  "device_name": "一号路灯",
  "location": "校门口",
  "status": "offline",
  "id": 1
}
```

错误情况：

- `400`：设备编码已存在。

### 查询设备列表

响应：

```json
[
  {
    "device_code": "SL-001",
    "device_name": "一号路灯",
    "location": "校门口",
    "status": "offline",
    "id": 1
  }
]
```

### 查询单个设备

响应：

```json
{
  "device_code": "SL-001",
  "device_name": "一号路灯",
  "location": "校门口",
  "status": "offline",
  "id": 1
}
```

错误情况：

- `404`：设备不存在。

### 修改设备

PUT 支持只传需要修改的字段。

请求：

```json
{
  "device_name": "一号路灯-东门",
  "location": "东门入口",
  "status": "online"
}
```

响应：

```json
{
  "device_code": "SL-001",
  "device_name": "一号路灯-东门",
  "location": "东门入口",
  "status": "online",
  "id": 1
}
```

错误情况：

- `404`：设备不存在。
- `400`：修改后的设备编码与其他设备重复。

### 删除设备

响应：

```json
{
  "message": "删除成功"
}
```

错误情况：

- `404`：设备不存在。
