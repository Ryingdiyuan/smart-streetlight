# 接口设计

## 基础约定

- Base URL：`/api`
- 数据格式：JSON
- 时间格式：`YYYY-MM-DD HH:mm:ss`
- 当前阶段只设计接口，不要求全部实现。

## 通用接口

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/health` | 健康检查 |

## 设备管理

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/devices` | 查询设备列表 |
| GET | `/devices/{id}` | 查询设备详情 |
| POST | `/devices` | 新增设备 |
| PUT | `/devices/{id}` | 更新设备信息 |
| DELETE | `/devices/{id}` | 删除或解绑设备 |

设备响应字段初稿：

```json
{
  "id": 1,
  "deviceCode": "SL-001",
  "deviceName": "一号路灯",
  "location": "校门口",
  "status": "online"
}
```

## 光照数据

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/devices/{id}/latest-light` | 查询设备最新光照和路灯状态 |
| GET | `/devices/{id}/light-history` | 查询历史光照曲线数据 |

历史数据查询参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| startTime | string | 开始时间 |
| endTime | string | 结束时间 |
| interval | string | 聚合粒度，可选 `raw`、`minute`、`hour` |

## 阈值配置

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/devices/{id}/threshold` | 查询设备阈值配置 |
| PUT | `/devices/{id}/threshold` | 更新设备阈值配置 |

请求示例：

```json
{
  "lowThreshold": 100,
  "highThreshold": 300,
  "enabled": true
}
```

## 控制指令

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/devices/{id}/commands` | 手动下发开灯、关灯或亮度调节指令 |
| GET | `/devices/{id}/commands` | 查询设备控制日志 |

请求示例：

```json
{
  "command": "TURN_ON",
  "source": "manual"
}
```

## 告警管理

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/alarms` | 查询告警列表 |
| GET | `/alarms/{id}` | 查询告警详情 |
| PUT | `/alarms/{id}/handle` | 标记告警已处理 |

## 智能体问答

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/agent/chat` | 提交维护问题，返回智能体回答 |

请求示例：

```json
{
  "question": "SL-001 离线后应该怎么排查？",
  "deviceId": "SL-001"
}
```
