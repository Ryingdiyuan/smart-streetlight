# 接口设计

## 基础约定

- 后端基础地址：`http://127.0.0.1:8000`
- API 前缀：`/api`
- 交互格式：JSON
- 接口文档：`http://127.0.0.1:8000/docs`
- 前端本地开发地址：`http://localhost:5173` 或 `http://127.0.0.1:5173`

当前 CORS 配置已经允许以上两个前端本地开发地址。

## 通用接口

### GET /

用途：检查后端根路径是否运行。

响应示例：

```json
{
  "name": "智慧路灯节能系统后端",
  "version": "0.1.0",
  "status": "running"
}
```

### GET /api/health

用途：健康检查。

响应示例：

```json
{
  "status": "ok"
}
```

## 设备管理

设备数据保存到 MySQL 的 `devices` 表。

### GET /api/devices

用途：查询设备列表。

请求参数：无。

响应示例：

```json
[
  {
    "id": 1,
    "device_code": "SL-001",
    "device_name": "一号路灯",
    "location": "校门口",
    "status": "online",
    "last_heartbeat_at": "2026-07-03T10:00:00"
  }
]
```

### GET /api/devices/{id}

用途：查询单个设备详情。

路径参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| id | integer | 设备 ID |

常见错误：

- `404`：设备不存在。

### POST /api/devices

用途：新增设备。

请求体示例：

```json
{
  "device_code": "SL-001",
  "device_name": "一号路灯",
  "location": "校门口",
  "status": "offline"
}
```

响应示例：

```json
{
  "id": 1,
  "device_code": "SL-001",
  "device_name": "一号路灯",
  "location": "校门口",
  "status": "offline",
  "last_heartbeat_at": null
}
```

常见错误：

- `400`：设备编码已存在。

### PUT /api/devices/{id}

用途：修改设备信息，支持只传部分字段。

请求体示例：

```json
{
  "device_name": "一号路灯-东门",
  "location": "东门入口",
  "status": "online"
}
```

常见错误：

- `400`：修改后的设备编码已存在。
- `404`：设备不存在。

### DELETE /api/devices/{id}

用途：删除设备。

响应示例：

```json
{
  "message": "删除成功"
}
```

常见错误：

- `404`：设备不存在。

## 光照数据

光照数据保存到 MySQL 的 `light_data` 表。HTTP 上报和 MQTT telemetry 上报都会写入该表。

### POST /api/devices/{device_id}/light-data

用途：HTTP 模拟设备上报光照数据。

路径参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| device_id | integer | 设备 ID |

请求体示例：

```json
{
  "light_intensity": 120,
  "lamp_status": "off",
  "voltage": 220.5,
  "reported_at": "2026-07-03T10:00:00"
}
```

响应示例：

```json
{
  "id": 1,
  "device_id": 1,
  "light_intensity": 120,
  "lamp_status": "off",
  "voltage": 220.5,
  "reported_at": "2026-07-03T10:00:00",
  "created_at": "2026-07-03T10:00:01",
  "suggested_action": "TURN_ON"
}
```

说明：

- `suggested_action` 是自动判断建议值，不会在 Day 9 改成真实自动控制。
- 常见返回值：`TURN_ON`、`TURN_OFF`、`KEEP`。

常见错误：

- `404`：设备不存在。

### GET /api/devices/{device_id}/latest-light

用途：查询某个设备最新光照数据。

常见错误：

- `404`：设备不存在。
- `404`：暂无光照数据。

### GET /api/devices/{device_id}/light-history

用途：查询某个设备历史光照数据。

查询参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| limit | integer | 否 | 返回条数，默认 `20`，范围 `1-200` |
| start_time | datetime | 否 | 开始时间 |
| end_time | datetime | 否 | 结束时间 |

响应示例：

```json
[
  {
    "id": 1,
    "device_id": 1,
    "light_intensity": 120,
    "lamp_status": "off",
    "voltage": 220.5,
    "reported_at": "2026-07-03T10:00:00",
    "created_at": "2026-07-03T10:00:01"
  }
]
```

## 阈值配置

阈值配置保存到 MySQL 的 `threshold_configs` 表。设备存在但没有配置时，后端会自动创建默认配置。

### GET /api/devices/{device_id}/threshold

用途：查询设备阈值配置。

响应示例：

```json
{
  "id": 1,
  "device_id": 1,
  "low_threshold": 100,
  "high_threshold": 300,
  "enabled": true,
  "updated_at": "2026-07-03T10:00:00"
}
```

常见错误：

- `404`：设备不存在。

### PUT /api/devices/{device_id}/threshold

用途：更新设备阈值配置。

请求体示例：

```json
{
  "low_threshold": 100,
  "high_threshold": 300,
  "enabled": true
}
```

常见错误：

- `400`：`low_threshold` 必须小于 `high_threshold`。
- `404`：设备不存在。

## 控制命令

控制日志保存到 MySQL 的 `control_logs` 表。`MQTT_ENABLED=true` 时，后端向 MQTT command topic 发布控制命令。

### POST /api/devices/{device_id}/commands

用途：下发手动控制命令。

请求体示例：

```json
{
  "command": "TURN_ON"
}
```

```json
{
  "command": "SET_BRIGHTNESS",
  "brightness": 80
}
```

响应示例：

```json
{
  "id": 1,
  "device_id": 1,
  "command": "TURN_ON",
  "source": "manual",
  "result": "success",
  "request_payload": {
    "command": "TURN_ON",
    "source": "manual",
    "timestamp": "2026-07-03 10:00:00"
  },
  "reply_payload": null,
  "created_at": "2026-07-03T10:00:00"
}
```

说明：

- `command` 只允许：`TURN_ON`、`TURN_OFF`、`SET_BRIGHTNESS`。
- `SET_BRIGHTNESS` 必须传 `brightness`。
- `brightness` 范围为 `0-100`。
- `MQTT_ENABLED=false` 时不发布 MQTT，日志结果为 `skipped`。
- MQTT 发布失败时日志结果为 `failed`，接口不应崩溃。

常见错误：

- `400`：不支持的控制命令。
- `400`：亮度参数缺失或范围不合法。
- `404`：设备不存在。

### GET /api/devices/{device_id}/commands

用途：查询设备控制日志。

查询参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| limit | integer | 否 | 返回条数，默认 `20`，范围 `1-200` |

常见错误：

- `404`：设备不存在。

## 告警管理

告警日志保存到 MySQL 的 `alarm_logs` 表。当前主要记录设备离线告警。

### GET /api/alarms

用途：查询告警列表。

查询参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| handled | boolean | 否 | 按是否处理过滤 |
| alarm_type | string | 否 | 按告警类型过滤，例如 `offline` |
| limit | integer | 否 | 返回条数，默认 `20`，范围 `1-200` |

响应示例：

```json
[
  {
    "id": 1,
    "device_id": 1,
    "alarm_type": "offline",
    "alarm_level": "warning",
    "alarm_content": "设备 SL-001 已离线",
    "handled": false,
    "handled_at": null,
    "created_at": "2026-07-03T10:05:00"
  }
]
```

### GET /api/alarms/{alarm_id}

用途：查询告警详情。

常见错误：

- `404`：告警不存在。

### PUT /api/alarms/{alarm_id}/handle

用途：标记告警已处理。

响应示例：

```json
{
  "id": 1,
  "device_id": 1,
  "alarm_type": "offline",
  "alarm_level": "warning",
  "alarm_content": "设备 SL-001 已离线",
  "handled": true,
  "handled_at": "2026-07-03T10:10:00",
  "created_at": "2026-07-03T10:05:00"
}
```

常见错误：

- `404`：告警不存在。

## MQTT 相关接口行为

### Telemetry

后端订阅：

```text
streetlight/+/telemetry
```

设备发布示例：

```text
streetlight/SL-001/telemetry
```

```json
{
  "deviceId": "SL-001",
  "lightIntensity": 120,
  "lampStatus": "off",
  "voltage": 220.5,
  "timestamp": "2026-07-03 10:00:00"
}
```

### Status

后端订阅：

```text
streetlight/+/status
```

设备发布示例：

```text
streetlight/SL-001/status
```

```json
{
  "deviceId": "SL-001",
  "online": true,
  "lampStatus": "off",
  "timestamp": "2026-07-03 10:00:00"
}
```

### Command

后端发布：

```text
streetlight/{deviceCode}/command
```

设备可订阅：

```text
streetlight/SL-001/command
```

## 统一错误码说明

| 状态码 | 说明 | 常见场景 |
| --- | --- | --- |
| 400 | 请求参数错误 | `device_code` 重复、非法 `command`、阈值范围不合法、亮度范围不合法 |
| 404 | 资源不存在 | 设备不存在、告警不存在、暂无光照数据 |
| 422 | 请求体格式不符合 schema | JSON 字段类型错误、必填字段缺失 |
| 500 | 后端服务异常 | 数据库连接异常、数据库表结构未同步、未预期异常 |

说明：Day 9 只整理文档，不为了统一错误码大规模重构已有接口。
