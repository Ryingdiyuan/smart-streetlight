| 方法 | 路径 | 说明 | 当前状态 |

## 光照数据

光照数据保存到 MySQL 的 `light_data` 表。Day 4 支持 HTTP 模拟上报，Day 5 新增 MQTT telemetry 写入，HTTP 上报接口继续保留作为备用测试接口。

### 光照数据字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | integer | 光照数据 ID |
| device_id | integer | 关联设备 ID |
| light_intensity | integer | 光照强度，例如 `120` |
| lamp_status | string | 路灯状态，例如 `on`、`off` |
| voltage | number | 电压，可为空，例如 `220.5` |
| reported_at | string | 设备上报时间 |
| created_at | string | 后端入库时间 |

### 接口列表

| 方法 | 路径 | 说明 | 当前状态 |
| --- | --- | --- | --- |
| POST | `/devices/{device_id}/light-data` | HTTP 模拟设备上报光照数据 | 已实现 |
| GET | `/devices/{device_id}/latest-light` | 查询某个设备最新光照数据 | 已实现 |
| GET | `/devices/{device_id}/light-history` | 查询某个设备历史光照数据 | 已实现 |

### HTTP 模拟上报光照数据

请求：

```json
{
  "light_intensity": 120,
  "lamp_status": "off",
  "voltage": 220.5,
  "reported_at": "2026-07-03 10:00:00"
}
```

说明：

- `reported_at` 可不传，不传时后端使用当前时间。
- `device_id` 必须对应已存在的设备。

响应：

```json
{
  "id": 1,
  "device_id": 1,
  "light_intensity": 120,
  "lamp_status": "off",
  "voltage": 220.5,
  "reported_at": "2026-07-03T10:00:00",
  "created_at": "2026-07-03T10:00:01"
}
```

错误情况：

- `404`：设备不存在。

### 查询最新光照数据

接口：

```text
GET /api/devices/{device_id}/latest-light
```

说明：

- 按 `reported_at` 倒序查询最新一条数据。
- 设备不存在返回 `404`。
- 设备存在但暂无光照数据返回 `404`。

### 查询历史光照数据

接口：

```text
GET /api/devices/{device_id}/light-history
```

查询参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| limit | integer | 返回条数，默认 `20`，范围 `1-200` |
| start_time | string | 可选，开始时间 |
| end_time | string | 可选，结束时间 |

说明：

- 按 `reported_at` 倒序返回，最新数据排在前面。
- 设备不存在返回 `404`。
- 设备存在但暂无数据时返回空数组 `[]`。
