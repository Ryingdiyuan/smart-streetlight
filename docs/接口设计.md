# 接口设计

## 基础约定

- Base URL：`/api`
- 数据格式：JSON
- 时间格式：接口响应中使用 FastAPI 默认的 ISO 时间格式
- 当前阶段：已实现设备管理、光照数据、阈值配置、手动控制、告警查询接口

| 方法 | 路径 | 说明 | 当前状态 |

## 设备管理

设备数据保存到 MySQL 的 `devices` 表。

| 方法 | 路径 | 说明 | 当前状态 |
| --- | --- | --- | --- |
| GET | `/devices` | 查询设备列表 | 已实现 |
| GET | `/devices/{id}` | 查询单个设备详情 | 已实现 |
| POST | `/devices` | 新增设备 | 已实现 |
| PUT | `/devices/{id}` | 修改设备信息，支持部分字段 | 已实现 |
| DELETE | `/devices/{id}` | 删除设备 | 已实现 |

设备响应示例：

```json
{
  "id": 1,
  "device_code": "SL-001",
  "device_name": "一号路灯",
  "location": "校门口",
  "status": "online",
  "last_heartbeat_at": "2026-07-03T10:00:00"
}
```

## 光照数据

光照数据保存到 MySQL 的 `light_data` 表。HTTP 和 MQTT 上报都会写入该表。

| 方法 | 路径 | 说明 | 当前状态 |
| --- | --- | --- | --- |
| POST | `/devices/{device_id}/light-data` | HTTP 模拟设备上报光照数据 | 已实现 |
| GET | `/devices/{device_id}/latest-light` | 查询某个设备最新光照数据 | 已实现 |
| GET | `/devices/{device_id}/light-history` | 查询某个设备历史光照数据 | 已实现 |

## 阈值配置

阈值配置保存到 MySQL 的 `threshold_configs` 表。设备存在但没有配置时，后端会自动创建默认配置。

| 方法 | 路径 | 说明 | 当前状态 |
| --- | --- | --- | --- |
| GET | `/devices/{device_id}/threshold` | 查询设备阈值配置 | 已实现 |
| PUT | `/devices/{device_id}/threshold` | 更新设备阈值配置 | 已实现 |

## 手动控制

控制日志保存到 MySQL 的 `control_logs` 表。`MQTT_ENABLED=true` 时，后端会向 MQTT command topic 发布控制命令。

| 方法 | 路径 | 说明 | 当前状态 |
| --- | --- | --- | --- |
| POST | `/devices/{device_id}/commands` | 下发控制命令 | 已实现 |
| GET | `/devices/{device_id}/commands` | 查询设备控制日志 | 已实现 |

下发控制命令请求示例：

```json
{
  "command": "TURN_ON"
}
```

```json
{
  "command": "SET_BRIGHTNESS",
  "brightness": 80
}
```

说明：

- `command` 只允许：`TURN_ON`、`TURN_OFF`、`SET_BRIGHTNESS`。
- `SET_BRIGHTNESS` 必须传 `brightness`。
- `brightness` 范围为 `0-100`。
- `MQTT_ENABLED=false` 时不发布 MQTT，日志结果为 `skipped`。

## 告警管理

告警日志保存到 MySQL 的 `alarm_logs` 表。Day 8 主要实现设备离线告警。

| 方法 | 路径 | 说明 | 当前状态 |
| --- | --- | --- | --- |
| GET | `/alarms` | 查询告警列表 | 已实现 |
| GET | `/alarms/{alarm_id}` | 查询告警详情 | 已实现 |
| PUT | `/alarms/{alarm_id}/handle` | 标记告警已处理 | 已实现 |

### 查询告警列表

接口：

```text
GET /api/alarms
```

查询参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| handled | boolean | 可选，按是否处理过滤 |
| alarm_type | string | 可选，按告警类型过滤，例如 `offline` |
| limit | integer | 返回条数，默认 `20`，范围 `1-200` |

响应示例：

```json
[
  {
    "id": 1,
    "device_id": 1,
    "alarm_type": "offline",
    "alarm_level": "warning",
    "alarm_content": "设备 SL-001 已离线",
    "handled": false,
    "handled_at": null,
    "created_at": "2026-07-03T10:05:00"
  }
]
```

### 查询告警详情

接口：

```text
GET /api/alarms/1
```

说明：

- 告警不存在时返回 `404`。

### 标记告警已处理

接口：

```text
PUT /api/alarms/1/handle
```

说明：

- 设置 `handled=true`。
- 设置 `handled_at` 为当前后端时间。
- 告警不存在时返回 `404`。

## MQTT telemetry

后端订阅 `streetlight/+/telemetry`，收到合法数据后写入 `light_data`，并执行自动判断逻辑。

## MQTT status

后端订阅 `streetlight/+/status`，收到设备心跳后更新 `devices.status` 和 `devices.last_heartbeat_at`。
