## 光照数据

光照数据保存到 MySQL 的 `light_data` 表。HTTP 和 MQTT 上报都会写入该表。Day 6 写入后会执行自动判断逻辑。

| 方法 | 路径 | 说明 | 当前状态 |
| --- | --- | --- | --- |
| POST | `/devices/{device_id}/light-data` | HTTP 模拟设备上报光照数据 | 已实现 |
| GET | `/devices/{device_id}/latest-light` | 查询某个设备最新光照数据 | 已实现 |
| GET | `/devices/{device_id}/light-history` | 查询某个设备历史光照数据 | 已实现 |

### HTTP 模拟上报光照数据

请求：

```json
{
  "light_intensity": 120,
  "lamp_status": "off",
  "voltage": 220.5,
  "reported_at": "2026-07-03 10:00:00"
}
```

响应：

```json
{
  "id": 1,
  "device_id": 1,
  "light_intensity": 120,
  "lamp_status": "off",
  "voltage": 220.5,
  "reported_at": "2026-07-03T10:00:00",
  "created_at": "2026-07-03T10:00:01",
  "suggested_action": "KEEP"
}
```

`suggested_action` 取值：

| 值 | 说明 |
| --- | --- |
| TURN_ON | 光照低于低阈值，建议开灯 |
| TURN_OFF | 光照高于高阈值，建议关灯 |
| KEEP | 光照处于阈值范围内，保持不变 |
| DISABLED | 自动判断未启用 |

## 阈值配置

阈值配置保存到 MySQL 的 `threshold_configs` 表。设备存在但没有配置时，后端会自动创建默认配置。

默认配置：

```json
{
  "low_threshold": 100,
  "high_threshold": 300,
  "enabled": true
}
```

| 方法 | 路径 | 说明 | 当前状态 |
| --- | --- | --- | --- |
| GET | `/devices/{device_id}/threshold` | 查询设备阈值配置 | 已实现 |
| PUT | `/devices/{device_id}/threshold` | 更新设备阈值配置 | 已实现 |

### 查询阈值配置

响应：

```json
{
  "id": 1,
  "device_id": 1,
  "low_threshold": 100,
  "high_threshold": 300,
  "enabled": true,
  "updated_at": "2026-07-03T10:00:00"
}
```

### 修改阈值配置

请求：

```json
{
  "low_threshold": 80,
  "high_threshold": 260,
  "enabled": true
}
```

说明：

- `device_id` 必须对应已存在设备。
- `low_threshold` 必须小于 `high_threshold`。
- 如果 `low_threshold >= high_threshold`，返回 `400`。
- 设备不存在时返回 `404`。
