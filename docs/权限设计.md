# 权限设计

## 为什么需要登录和权限控制

智慧路灯系统后续会接入登录页面和不同身份用户。后端需要先提供认证基础能力，避免所有人都能执行设备管理、控制命令、告警处理等操作。

Day 13 只完成后端认证基础：

1. 用户表 `users`。
2. 登录接口。
3. JWT Token 生成和校验。
4. 角色字段。
5. 权限依赖函数。

本轮没有给 Day 1 到 Day 12 的旧业务接口强制加鉴权，避免破坏已有演示链路。

## 用户角色设计

| 角色 | 说明 | 后续权限方向 |
| --- | --- | --- |
| `admin` | 管理员 | 用户管理、系统配置、设备管理、告警处理、控制命令 |
| `operator` | 运维人员 | 查看数据、下发控制命令、处理告警 |
| `viewer` | 只读用户 | 查看设备、光照、告警和智能体问答 |

## 权限依赖函数

后端提供以下依赖函数，供后续逐步接入业务接口：

1. `get_current_user`：解析 `Authorization: Bearer <token>`，返回当前用户。
2. `require_admin`：要求 `role=admin`。
3. `require_operator_or_admin`：允许 `admin`、`operator`。
4. `require_viewer_or_above`：允许 `admin`、`operator`、`viewer`。

Day 13 只提供这些函数，不强制修改旧业务接口。

## 登录认证流程

1. 用户输入账号密码。
2. 前端调用 `POST /api/auth/login`。
3. 后端查询用户并校验密码哈希。
4. 校验成功后返回 JWT access token。
5. 前端保存 token。
6. 后续需要鉴权的请求携带：

```text
Authorization: Bearer <token>
```

7. 后端解析 token，识别用户身份和角色。

## 后端接口说明

### POST /api/auth/init-admin

用途：初始化第一个管理员。

请求示例：

```json
{
  "username": "admin",
  "password": "123456"
}
```

响应示例：

```json
{
  "id": 1,
  "username": "admin",
  "role": "admin",
  "is_active": true
}
```

规则：

- 仅当 `users` 表为空时可调用。
- 已存在任何用户后再次调用返回 `400`。
- 密码只保存哈希，不保存明文。
- 响应不返回 `password_hash`。

### POST /api/auth/login

用途：用户登录。

请求示例：

```json
{
  "username": "admin",
  "password": "123456"
}
```

响应示例：

```json
{
  "access_token": "jwt_token_string",
  "token_type": "bearer",
  "user": {
    "id": 1,
    "username": "admin",
    "role": "admin"
  }
}
```

错误：

- 用户不存在或密码错误：`401`。
- 用户被禁用：`403`。

### GET /api/auth/me

用途：获取当前登录用户。

Header：

```text
Authorization: Bearer <token>
```

响应示例：

```json
{
  "id": 1,
  "username": "admin",
  "role": "admin",
  "is_active": true
}
```

错误：

- Token 缺失、无效、过期或用户不存在：`401`。
- 用户被禁用：`403`。

## 初始化管理员方式

1. 确认 MySQL 已启动。
2. 启动后端。
3. 调用 `POST /api/auth/init-admin`。
4. 再调用 `POST /api/auth/login` 获取 token。
5. 调用 `GET /api/auth/me` 验证 token 是否有效。

## 前端接入说明

前端登录页面完成后，可按以下流程接入：

1. 登录表单提交用户名和密码到 `POST /api/auth/login`。
2. 登录成功后保存 `access_token`。
3. 后续需要鉴权的请求添加 Header：

```text
Authorization: Bearer <token>
```

4. 进入页面时可调用 `GET /api/auth/me` 获取当前用户信息和角色。

Day 13 不写前端代码，也不要求现有前端页面立即接入。

## 安全注意事项

1. `.env` 不要提交到 GitHub。
2. 真实 `JWT_SECRET_KEY` 不要写入 README、docs 或 `.env.example`。
3. 密码不明文入库，只保存 `password_hash`。
4. Token 中不存储密码、密码哈希、API Key。
5. 日志中不要打印密码、Token、API Key 或真实 JWT Secret。
