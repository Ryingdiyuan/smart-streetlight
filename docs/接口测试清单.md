# 接口测试清单

## 测试前准备

1. MySQL 已启动。
2. `smart_streetlight` 数据库已创建。
3. 后端 `.env` 配置正确。
4. 如果已有旧版 `devices` 表，已执行：

```sql
ALTER TABLE devices ADD COLUMN last_heartbeat_at DATETIME NULL;
```

5. 后端已启动：

```powershell
cd D:\code\smart-streetlight\backend
.\.venv\Scripts\uvicorn.exe app.main:app --reload
```

## 1. 健康检查

| 测试项 | 预期 |
| --- | --- |
| `GET /` | 返回系统名称、版本、运行状态 |
| `GET /api/health` | 返回 `{"status": "ok"}` |

## 2. 设备管理

| 测试项 | 预期 |
| --- | --- |
| `POST /api/devices` | 成功创建设备 |
| `GET /api/devices` | 返回设备列表 |
| `GET /api/devices/{id}` | 返回设备详情 |
| `PUT /api/devices/{id}` | 成功修改设备 |
| `DELETE /api/devices/{id}` | 返回 `{"message": "删除成功"}` |
| 重复 `device_code` | 返回 `400` |
| 查询不存在设备 | 返回 `404` |

## 3. 光照数据

| 测试项 | 预期 |
| --- | --- |
| `POST /api/devices/{device_id}/light-data` | 成功写入光照数据，并返回 `suggested_action` |
| `GET /api/devices/{device_id}/latest-light` | 返回最新光照数据 |
| `GET /api/devices/{device_id}/light-history` | 返回历史光照数据列表 |
| 设备不存在时上报光照 | 返回 `404` |
| 暂无光照数据时查询最新光照 | 返回 `404` |

## 4. 阈值配置

| 测试项 | 预期 |
| --- | --- |
| `GET /api/devices/{device_id}/threshold` | 返回阈值配置；没有配置时自动创建默认配置 |
| `PUT /api/devices/{device_id}/threshold` | 成功更新阈值配置 |
| `low_threshold >= high_threshold` | 返回 `400` |
| `enabled=false` 后上报光照 | `suggested_action` 为 `DISABLED` |

## 5. 控制命令

| 测试项 | 预期 |
| --- | --- |
| `POST /api/devices/{device_id}/commands`，`TURN_ON` | 成功写入控制日志 |
| `POST /api/devices/{device_id}/commands`，`TURN_OFF` | 成功写入控制日志 |
| `POST /api/devices/{device_id}/commands`，`SET_BRIGHTNESS` | 成功写入控制日志 |
| `GET /api/devices/{device_id}/commands` | 返回控制日志列表 |
| 非法 `command` | 返回 `400` |
| `SET_BRIGHTNESS` 缺少 `brightness` | 返回 `400` |
| `brightness` 超出 `0-100` | 返回 `400` |

说明：

- `MQTT_ENABLED=false` 时控制日志结果为 `skipped`。
- `MQTT_ENABLED=true` 且 MQTT 发布成功时结果为 `success`。
- MQTT 发布失败时结果为 `failed`，接口不应崩溃。

## 6. 告警管理

| 测试项 | 预期 |
| --- | --- |
| `GET /api/alarms` | 返回告警列表 |
| `GET /api/alarms/{alarm_id}` | 返回告警详情 |
| `PUT /api/alarms/{alarm_id}/handle` | 标记告警为已处理 |
| 告警不存在 | 返回 `404` |

## 7. MQTT 测试

| 测试项 | Topic | 预期 |
| --- | --- | --- |
| telemetry 写入 `light_data` | `streetlight/{deviceCode}/telemetry` | 最新光照接口能查到数据 |
| status 更新设备状态 | `streetlight/{deviceCode}/status` | `GET /api/devices` 中 `status` 和 `last_heartbeat_at` 更新 |
| command 发布 | `streetlight/{deviceCode}/command` | MQTTX 能收到手动控制接口发布的命令 |

## 8. 智能体问答

| 测试项 | 预期 |
| --- | --- |
| `POST /api/agent/chat`，按 `device_id` 提问 | 返回设备维度运维建议 |
| `POST /api/agent/chat`，按 `device_code` 提问 | 返回设备维度运维建议 |
| `POST /api/agent/chat`，不指定设备 | 返回系统整体运维建议 |
| `LLM_ENABLED=false` | 返回 `source=mock` |
| LLM 配置缺失或调用失败 | 返回 `source=fallback`，接口不崩溃 |
| `question` 为空 | 返回 `400` |
| `question` 超过 500 字 | 返回 `400` |
| 设备不存在 | 返回 `404` |

## 9. 冒烟测试脚本

启动后端后运行：

```powershell
cd D:\code\smart-streetlight\backend
.\.venv\Scripts\python.exe scripts\smoke_test_api.py
```

脚本会使用唯一设备编码创建测试设备，不依赖固定 `id=1`，也不会删除数据库已有业务数据。
