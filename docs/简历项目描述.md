# 简历项目描述

## 简历版项目描述

智慧路灯节能系统是一个基于 FastAPI + MySQL + MQTT 的物联网后端项目，面向路灯设备管理、光照数据采集、节能阈值判断、手动控制、设备心跳和离线告警等场景。项目采用 FastAPI 提供 RESTful API，使用 SQLAlchemy 操作 MySQL，使用 Paho-MQTT 接入设备 telemetry/status 主题，并通过 APScheduler 定时检测设备离线状态。

## 技术栈关键词

- Python 3.10+
- FastAPI
- MySQL
- SQLAlchemy
- Pydantic
- Paho-MQTT
- APScheduler
- RESTful API
- MQTT
- Swagger / OpenAPI

## 个人负责内容

后端方向可描述为：

1. 搭建 FastAPI 后端项目结构，划分 `routers`、`models`、`schemas`、`services`、`mqtt`、`tasks`、`core` 等模块。
2. 设计并实现设备管理接口，支持设备新增、查询、修改和删除。
3. 使用 SQLAlchemy 设计设备、光照数据、阈值配置、控制日志和告警日志模型。
4. 实现 HTTP 光照数据上报、最新光照查询和历史光照查询接口。
5. 实现设备阈值配置和基于光照强度的自动判断逻辑，返回控制建议。
6. 接入 MQTT telemetry topic，将设备遥测数据写入 MySQL。
7. 实现手动控制接口，记录控制日志，并在 MQTT 启用时发布 command 消息。
8. 实现设备 status 心跳处理、离线检测和告警日志查询接口。
9. 编写接口设计、数据库设计、MQTT 主题设计、联调流程和测试清单等文档。
10. 编写后端 HTTP 冒烟测试脚本，便于本地快速验证核心接口。

## 面试时可讲的亮点

### 1. 分层结构清晰

项目按路由、模型、schema、service、MQTT、定时任务、核心配置拆分，后续扩展新模块时不需要大规模改动已有代码。

### 2. HTTP 和 MQTT 双通道

HTTP 用于前端管理和模拟上报，MQTT 用于设备遥测、心跳和控制消息，符合物联网项目常见通信方式。

### 3. 设备离线检测链路完整

设备通过 status topic 上报心跳，后端记录 `last_heartbeat_at`，定时任务根据阈值判断离线并写入 `alarm_logs`，形成可查询、可处理的告警闭环。

### 4. 注重联调和可验证性

项目不仅实现接口，还补充了接口设计、前端联调清单、后端联调流程、接口测试清单和冒烟测试脚本，方便小组成员协作。

### 5. 明确当前边界

当前版本没有登录鉴权、没有 Alembic 迁移、没有 command/reply 回执处理，智能体问答仍是后续扩展计划。面试时不要把这些计划中的功能说成已完成。
