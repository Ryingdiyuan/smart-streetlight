# 后端技术选型

## 技术栈

- Python 3.10+
- FastAPI
- MySQL
- SQLAlchemy
- Paho-MQTT
- APScheduler

## 为什么选择 Python + FastAPI

本项目需要在较短实训周期内完成“设备数据接入、后端接口、数据库存储、告警与联动、前端展示、智能体问答”的闭环。Python 适合快速实现数据处理、MQTT 模拟设备联调和智能体能力接入，团队学习成本较低，也便于后续编写 mock-device 脚本。

FastAPI 的优势是：

1. 启动快，代码结构轻量，适合先搭骨架、再逐步补业务。
2. 原生支持 OpenAPI 文档，后端接口可以直接给前端联调使用。
3. 基于 Pydantic 做请求和响应校验，接口字段更清晰。
4. 支持异步能力，后续可扩展 WebSocket、SSE 或异步 MQTT 处理。
5. 与 SQLAlchemy、APScheduler、Paho-MQTT 集成简单，适合智慧路灯这类“HTTP API + 设备消息 + 定时任务”的后端形态。

## 各组件职责

| 组件 | 职责 |
| --- | --- |
| FastAPI | 提供 REST API、健康检查、接口文档 |
| MySQL | 存储设备、光照数据、阈值、控制日志、告警日志 |
| SQLAlchemy | 数据模型和数据库访问 |
| Paho-MQTT | 订阅设备上报、发布控制指令 |
| APScheduler | 定时检测设备离线、后续可扩展定时报表或数据清理 |

## 当前阶段边界

当前只完成可运行骨架和设计文档，不实现完整业务功能。后续开发再逐步补充设备管理、数据入库、联动控制、告警处理和智能体接口。
