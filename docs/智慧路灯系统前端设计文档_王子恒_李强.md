# 智慧路灯节能系统前端设计文档（王子恒、李强分工版）

**项目名称：** 智慧路灯节能系统前端  
**文档版本：** V1.0  
**适用范围：** 前端开发、接口联调、课程答辩、项目展示  
**主要技术栈：** Vue 3、TypeScript、Vite、Vue Router、ECharts、vue-echarts、高德地图 JSAPI、Fetch API  
**主要负责人：** 王子恒、李强

## 一、项目概述

智慧路灯节能系统前端用于展示智慧路灯设备运行状态、实时光照数据、历史光照趋势、设备地图分布、告警日志、控制记录、用户权限和智能问答结果，是管理员、维修人员和普通用户访问系统功能的主要入口。

前端系统通过 HTTP JSON 接口与后端 FastAPI 服务通信，完成登录认证、设备管理、阈值配置、手动控制、告警处理、智能问答等业务操作；同时通过图表、状态卡片、地图和表格等方式，将设备运行数据可视化，便于项目演示和运维排查。

系统前端目标是形成一个完整的“登录认证—权限控制—设备总览—设备管理—实时监测—历史分析—告警处理—智能问答”的可视化管理闭环。

## 二、前端功能需求

### 2.1 登录认证功能

前端需要提供登录页面，用户输入账号和密码后调用后端登录接口，登录成功后保存 access token 和用户信息，并在后续请求中自动携带认证信息。

主要功能包括：

1. 用户名、密码输入与基础校验。
2. 调用 `/api/auth/login` 获取登录结果。
3. 将 token 和用户角色保存到 `localStorage`。
4. 未登录用户访问业务页面时自动跳转登录页。
5. 已登录用户访问登录页时自动跳转系统首页。
6. 请求返回 `401` 时清除会话并要求重新登录。

### 2.2 总览大屏功能

总览大屏用于展示系统整体运行情况，是进入系统后的默认页面。

主要功能包括：

1. 展示设备总数、在线设备数、未处理告警数、当前开灯数量。
2. 展示重点设备的最新状态和光照历史趋势。
3. 展示最近告警记录。
4. 展示最近控制命令记录。
5. 聚合设备、光照、告警、控制日志等多类接口数据。

### 2.3 设备管理功能

设备管理模块用于展示和维护智慧路灯设备基础信息。

主要功能包括：

1. 查询设备列表。
2. 查看设备详情。
3. 新增设备。
4. 编辑设备编码、名称、位置、经纬度等信息。
5. 展示设备在线/离线状态、灯光状态和最后心跳时间。
6. 支持进入详情页查看该设备的光照、阈值、控制日志和告警信息。

### 2.4 设备详情与阈值配置功能

设备详情页用于展示单台设备的完整运行信息，并支持阈值配置和手动控制。

主要功能包括：

1. 展示设备基础信息。
2. 展示当前光照强度和灯光状态。
3. 展示历史光照曲线。
4. 查询并展示设备阈值配置。
5. 修改低阈值、高阈值和自动判断开关。
6. 发送开灯、关灯等手动控制命令。
7. 展示控制日志和关联告警。

### 2.5 实时光照监测功能

实时光照监测页面用于集中展示所有设备当前光照数据。

主要功能包括：

1. 查询所有设备的最新光照数据。
2. 展示设备编码、名称、位置、在线状态、灯光状态、当前光照强度和更新时间。
3. 对暂无光照数据的设备做空值兜底展示。
4. 支持作为项目演示中验证设备数据上报效果的页面。

### 2.6 历史光照数据功能

历史光照页面用于查看单台或多台设备的历史光照趋势。

主要功能包括：

1. 查询设备历史光照数据。
2. 支持按设备查看光照强度变化。
3. 使用 ECharts 绘制趋势曲线。
4. 支持时间范围和数量限制等查询参数。
5. 辅助判断节能控制策略是否合理。

### 2.7 设备地图功能

设备地图用于展示路灯设备的空间分布。

主要功能包括：

1. 读取设备经纬度信息。
2. 使用高德地图 JSAPI 展示设备点位。
3. 按在线/离线状态区分设备显示。
4. 支持在地图中查看设备基础信息。
5. 辅助演示智慧路灯系统的 GIS 可视化能力。

### 2.8 告警日志功能

告警模块用于查看和处理设备离线等异常告警。

主要功能包括：

1. 查询告警列表。
2. 展示告警类型、等级、内容、所属设备、处理状态和创建时间。
3. 对未处理告警进行处理操作。
4. 处理成功后刷新页面数据。
5. 根据用户权限控制告警处理按钮显示。

### 2.9 用户管理与权限控制功能

系统前端根据用户角色控制页面访问和按钮显示。

主要功能包括：

1. 支持 `admin`、`maintainer`、`user` 三类前端角色。
2. 管理员可以访问全部页面和操作。
3. 维修人员可以查看设备、执行设备控制、处理告警和使用智能问答。
4. 普通用户可以查看总览、设备、光照数据和智能问答。
5. 用户管理页面仅管理员可见。
6. 无权限访问时跳转到无权限页面。

### 2.10 智能问答功能

智能问答页面用于向后端运维智能体提问，获取设备维护建议。

主要功能包括：

1. 展示内置推荐问题。
2. 输入运维问题并调用 `/api/agent/chat`。
3. 支持按设备 ID 或设备编码限定提问范围。
4. 展示后端返回的回答内容和关联设备信息。
5. 用于答辩时演示系统的数据分析和辅助运维能力。

### 2.11 模拟器与硬件联调前端功能

项目中除主系统前端 `frontend/` 外，还包含 `hardware-frontend/`，用于硬件模拟、设备上报和 MQTT 联调场景。

主要功能包括：

1. 查看 MQTT Broker 配置。
2. 配置模拟设备基础参数。
3. 启动或停止模拟设备。
4. 查看模拟设备运行状态和发布次数。
5. 查看模拟器日志。
6. 配合后端 MQTT 和 HTTP 接口完成无真实硬件情况下的演示。

## 三、系统总体架构设计

### 3.1 架构说明

前端采用 Vue 3 单页应用架构，整体分为页面层、布局层、组件层、路由权限层、服务层、数据模型层和样式主题层。

```text
用户浏览器
    ↓
Vue 3 单页应用
    ↓
Vue Router 路由与权限守卫
    ↓
页面 Pages + 公共组件 Components
    ↓
Service 业务服务封装
    ↓
HTTP 工具层 lib/http.ts
    ↓
FastAPI 后端接口 / Mock 数据
```

系统同时支持两种数据模式：

```text
VITE_SERVICE_MODE=api
    ↓
调用真实后端 /api 接口

VITE_SERVICE_MODE=mock
    ↓
调用前端 mock 服务数据
```

这种设计保证了前端在后端尚未完全就绪时可以独立开发和演示，在后端接口完成后又可以快速切换到真实联调模式。

#### 4.2.4 模块实现思路

登录模块调用 `/api/auth/login`，登录成功后将 token 和用户信息保存到 `localStorage`。请求封装位于 `lib/http.ts`，所有业务请求都会从本地会话中读取 token，并自动添加 `Authorization: Bearer <access_token>`。当接口返回 `401` 时，前端清除登录状态，避免继续使用失效 token。

权限模块位于 `permissions.ts`，前端定义 `admin`、`maintainer`、`user` 三种角色和对应权限。路由守卫在 `router/index.ts` 中实现，未登录用户会被重定向到登录页，无权限用户会被跳转到 `ForbiddenPage.vue`。页面内按钮也根据 `can(permission)` 判断是否显示，避免普通用户看到不应该操作的功能。

接口归一化模块位于 `normalizers.ts`。后端字段多为蛇形命名，例如 `device_code`、`last_heartbeat_at`；前端页面使用驼峰命名，例如 `deviceCode`、`lastHeartbeatAt`。归一化层负责统一转换字段、格式化时间、处理状态枚举，降低页面组件复杂度。

告警页面通过 `alarmService.ts` 查询 `/api/alarms`，并调用 `/api/alarms/{alarm_id}/handle` 完成告警处理。智能问答页面通过 `agentService.ts` 调用 `/api/agent/chat`，将用户问题发送给后端智能体并展示维护建议。

模拟器接口通过 `simulatorService.ts` 管理 MQTT 配置、模拟设备和模拟器日志，配合 `hardware-frontend` 支撑硬件联调和无真实设备演示。

## 七、接口封装设计

### 7.1 HTTP 工具层

`frontend/src/lib/http.ts` 对浏览器 `fetch` 进行了统一封装，提供：

1. `http.get<T>()`
2. `http.post<T>()`
3. `http.put<T>()`
4. `http.delete<T>()`

统一能力包括：

1. 拼接 `VITE_API_BASE_URL`。
2. 自动添加 `Content-Type: application/json`。
3. 自动添加 `Authorization: Bearer <access_token>`。
4. 统一解析后端返回的 `data`、`result`、`items` 包装结构。
5. 统一处理 `204` 空响应。
6. `401` 时自动清除登录状态。
7. 抛出携带 HTTP 状态码的错误，便于服务层判断。

### 7.2 API 与 Mock 切换

运行时配置位于 `frontend/src/config/env.ts`：

```ts
const serviceMode = import.meta.env.VITE_SERVICE_MODE ?? "api";

export const runtimeConfig = {
  apiBaseUrl: import.meta.env.VITE_API_BASE_URL ?? "/api",
  serviceMode: serviceMode === "api" ? "api" : "mock",
};
```

服务切换位于 `serviceRuntime.ts`：

```ts
export function createServiceSwitcher<T>(mockImpl: T, apiImpl: T): T {
  return (isMockServiceEnabled ? mockImpl : apiImpl) as T;
}
```

当 `VITE_SERVICE_MODE=mock` 时，页面读取前端 Mock 数据；当 `VITE_SERVICE_MODE=api` 时，页面调用真实后端接口。

### 7.3 字段归一化

后端接口字段多采用蛇形命名，前端采用驼峰命名。`normalizers.ts` 负责转换数据结构。

示例：

| 后端字段 | 前端字段 |
|---|---|
| `device_code` | `deviceCode` |
| `device_name` | `deviceName` |
| `last_heartbeat_at` | `lastHeartbeatAt` |
| `light_intensity` | `lightIntensity` |
| `lamp_status` | `lampStatus` |
| `created_at` | `createdAt` |

同时归一化层还负责：

1. 格式化时间字符串。
2. 统一设备在线状态。
3. 统一灯光开关状态。
4. 统一告警等级。
5. 统一控制命令执行结果。

## 十四、项目总结

本系统前端围绕智慧路灯节能管理场景，完成了登录认证、路由权限、总览大屏、设备管理、设备详情、实时光照、历史趋势、地图展示、告警处理、用户管理、智能问答和模拟器联调等功能。

前端采用 Vue 3 + TypeScript + Vite 技术栈，页面结构清晰，服务层统一封装接口调用，并通过字段归一化降低页面组件复杂度。系统同时支持真实 API 和 Mock 数据两种模式，便于前后端并行开发、接口联调和课程答辩演示。

两人分工采用按前端能力模块拆分的方式：

这种分工保证了前端系统既有完整的可视化展示能力，也具备稳定的接口联调、权限控制和异常处理能力。答辩时两名成员都可以围绕自己负责的模块讲清楚功能目标、核心代码、实现逻辑和演示流程。
