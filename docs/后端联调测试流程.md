# 后端联调测试流程

## 1. 启动前准备

### MySQL

确认 MySQL 已启动，并且数据库存在：

```sql
CREATE DATABASE IF NOT EXISTS smart_streetlight
DEFAULT CHARSET utf8mb4
COLLATE utf8mb4_unicode_ci;
```

如果 Day 8 前已经创建过旧版 `devices` 表，需要手动补充心跳字段。`create_all` 不会自动给已有表新增字段。

```sql
ALTER TABLE devices ADD COLUMN last_heartbeat_at DATETIME NULL;
```

如果不确定字段是否存在，先执行：

```sql
SHOW COLUMNS FROM devices LIKE 'last_heartbeat_at';
```

### MQTT Broker

确认 Mosquitto Broker 已启动：

- Host：`127.0.0.1`
- Port：`1883`

Windows 可用以下命令检查端口：

```powershell
netstat -ano | findstr :1883
```

### .env 配置

确认 `backend/.env` 中至少包含：

```env
MYSQL_HOST=127.0.0.1
MYSQL_PORT=3307
MYSQL_USER=root
MYSQL_PASSWORD=你的数据库密码
MYSQL_DATABASE=smart_streetlight

MQTT_ENABLED=true
MQTT_HOST=127.0.0.1
MQTT_PORT=1883

SCHEDULER_ENABLED=true
DEVICE_OFFLINE_SECONDS=180
```

如果只测试 HTTP 接口，可以先设置：

```env
MQTT_ENABLED=false
```

## 2. 启动后端

进入后端目录：

```powershell
cd D:\code\smart-streetlight\backend
```

启动服务：

```powershell
.\.venv\Scripts\uvicorn.exe app.main:app --reload
```

启动后访问：

```text
http://127.0.0.1:8000/docs
```

## 3. 推荐测试顺序

1. 健康检查：`GET /api/health`
2. 创建设备：`POST /api/devices`
3. 查询设备列表：`GET /api/devices`
4. 查询设备详情：`GET /api/devices/{id}`
5. HTTP 上报光照：`POST /api/devices/{device_id}/light-data`
6. 查询最新光照：`GET /api/devices/{device_id}/latest-light`
7. 查询历史光照：`GET /api/devices/{device_id}/light-history`
8. 查询阈值配置：`GET /api/devices/{device_id}/threshold`
9. 修改阈值配置：`PUT /api/devices/{device_id}/threshold`
10. 再次 HTTP 上报光照，观察 `suggested_action`
11. MQTTX 发布 telemetry，确认能写入 `light_data`
12. MQTTX 订阅 command topic
13. 调用手动控制接口：`POST /api/devices/{device_id}/commands`
14. MQTTX 发布 status 心跳
15. 查询告警：`GET /api/alarms`
16. 标记告警已处理：`PUT /api/alarms/{alarm_id}/handle`

## 4. PowerShell 测试命令

以下命令适合 Windows PowerShell，使用 `Invoke-RestMethod`。

### 冒烟测试脚本

如果只想快速验证核心 HTTP 接口，可以先启动后端，再执行：

```powershell
cd D:\code\smart-streetlight\backend
.\.venv\Scripts\python.exe scripts\smoke_test_api.py
```

脚本会依次测试健康检查、设备创建、设备查询、光照上报、阈值配置、手动控制和告警列表。测试设备使用 `TEST-SL-时间戳` 编码，不依赖固定 `id=1`，也不会删除数据库已有业务数据。

### 健康检查

```powershell
Invoke-RestMethod -Method Get -Uri "http://127.0.0.1:8000/api/health"
```

### 创建设备

```powershell
$device = @{
  device_code = "SL-001"
  device_name = "一号路灯"
  location = "校门口"
  status = "offline"
} | ConvertTo-Json

Invoke-RestMethod -Method Post -Uri "http://127.0.0.1:8000/api/devices" -ContentType "application/json" -Body $device
```

### 查询设备列表

```powershell
Invoke-RestMethod -Method Get -Uri "http://127.0.0.1:8000/api/devices"
```

### HTTP 上报光照

```powershell
$lightData = @{
  light_intensity = 120
  lamp_status = "off"
  voltage = 220.5
  reported_at = "2026-07-03T10:00:00"
} | ConvertTo-Json

Invoke-RestMethod -Method Post -Uri "http://127.0.0.1:8000/api/devices/1/light-data" -ContentType "application/json" -Body $lightData
```

### 查询最新光照

```powershell
Invoke-RestMethod -Method Get -Uri "http://127.0.0.1:8000/api/devices/1/latest-light"
```

### 查询历史光照

```powershell
Invoke-RestMethod -Method Get -Uri "http://127.0.0.1:8000/api/devices/1/light-history?limit=20"
```

### 查询阈值配置

```powershell
Invoke-RestMethod -Method Get -Uri "http://127.0.0.1:8000/api/devices/1/threshold"
```

### 修改阈值配置

```powershell
$threshold = @{
  low_threshold = 100
  high_threshold = 300
  enabled = $true
} | ConvertTo-Json

Invoke-RestMethod -Method Put -Uri "http://127.0.0.1:8000/api/devices/1/threshold" -ContentType "application/json" -Body $threshold
```

### 手动控制

```powershell
$command = @{
  command = "TURN_ON"
} | ConvertTo-Json

Invoke-RestMethod -Method Post -Uri "http://127.0.0.1:8000/api/devices/1/commands" -ContentType "application/json" -Body $command
```

### 查询控制日志

```powershell
Invoke-RestMethod -Method Get -Uri "http://127.0.0.1:8000/api/devices/1/commands?limit=20"
```

### 查询告警

```powershell
Invoke-RestMethod -Method Get -Uri "http://127.0.0.1:8000/api/alarms?limit=20"
```

### 标记告警已处理

```powershell
Invoke-RestMethod -Method Put -Uri "http://127.0.0.1:8000/api/alarms/1/handle"
```

## 5. MQTTX 测试说明

### 连接配置

- Host：`127.0.0.1`
- Port：`1883`
- Username：如果 Mosquitto 未配置认证则留空
- Password：如果 Mosquitto 未配置认证则留空

### Telemetry topic

发布主题：

```text
streetlight/SL-001/telemetry
```

Payload：

```json
{
  "deviceId": "SL-001",
  "lightIntensity": 120,
  "lampStatus": "off",
  "voltage": 220.5,
  "timestamp": "2026-07-03 10:00:00"
}
```

验证方式：

1. 发布 telemetry。
2. 调用 `GET /api/devices/1/latest-light`。
3. 如果能查到最新数据，说明 MQTT telemetry 入库成功。

### Status topic

发布主题：

```text
streetlight/SL-001/status
```

Payload：

```json
{
  "deviceId": "SL-001",
  "online": true,
  "lampStatus": "off",
  "timestamp": "2026-07-03 10:00:00"
}
```

验证方式：

1. 发布 status。
2. 调用 `GET /api/devices`。
3. 查看设备 `status` 和 `last_heartbeat_at` 是否更新。

### Command topic

订阅主题：

```text
streetlight/SL-001/command
```

然后调用：

```text
POST /api/devices/1/commands
```

如果 `MQTT_ENABLED=true` 且 Broker 正常，MQTTX 应该收到后端发布的控制命令。

## 6. 常见问题

### 1883 端口被占用

现象：Mosquitto 无法启动，或后端连接 MQTT Broker 失败。

检查：

```powershell
netstat -ano | findstr :1883
```

### .env 没生效

确认当前启动目录是：

```text
D:\code\smart-streetlight\backend
```

因为配置类读取的是后端目录下的 `.env`。

### MySQL 连接失败

检查：

1. MySQL 是否启动。
2. `MYSQL_PORT` 是否为本机实际端口，例如 `3307`。
3. `MYSQL_PASSWORD` 是否正确。
4. `smart_streetlight` 数据库是否存在。

### create_all 不会自动给旧表加字段

如果报错：

```text
Unknown column 'devices.last_heartbeat_at'
```

执行：

```sql
ALTER TABLE devices ADD COLUMN last_heartbeat_at DATETIME NULL;
```

### 查询不到设备

MQTT payload 中的 `deviceId` 必须等于数据库中的 `devices.device_code`，例如：

```text
SL-001
```

### MQTTX 发布 topic 不能为空

发布 MQTT 消息时必须填写 topic，例如：

```text
streetlight/SL-001/telemetry
```

或者：

```text
streetlight/SL-001/status
```
